Коннекторы и MCP-протокол

Как агент получает доступ к внешним сервисам, вызывает инструменты и остаётся в границах выданных разрешений.

Агенту нужны не только «мозги», но и подключённые руки.

1. Коннектор

Коннектор — это доступ к конкретному сервису или данным: почта, диск, календарь, репозиторий, база, CRM.

  • Куда: какой сервис подключаем.
  • Что можно: читать, писать, искать, создавать, изменять.
  • Под чьим доступом: личный аккаунт, команда, отдельный токен.

2. MCP

MCP (Model Context Protocol) — универсальная «розетка» для внешних цифровых инструментов и источников данных.

  • Один формат: агенту не нужно заново учиться каждому сервису.
  • Инструменты: поиск, файлы, базы, задачи, документы.
  • Контекст: данные приходят агенту в понятной структуре.

3. Tool use / function calling

Tool use — это само действие: модель не просто говорит «надо открыть файл», а вызывает инструмент и получает результат обратно.

Промпт → LLM → выбор инструмента → вызов → наблюдение → следующий шаг
LLM без tool use Советует, объясняет, пишет текст.
LLM с tool use Вызывает поиск, файл, код или другое действие.
MCP Стандартная розетка, через которую часть инструментов подключается.

4. Не всё идёт через MCP

Веб-доступ, зрение и управление компьютером могут быть отдельными механизмами.

  • Веб: поиск и чтение страниц.
  • Зрение: понимание изображения или экрана.
  • Computer use: мышь, клавиатура, реальный интерфейс.

5. Разрешения

Коннекторы дают агенту ключи, но не отменяют границы.

  • Заранее: папки, сервисы, аккаунты, токены.
  • На красных линиях: отправить, опубликовать, удалить, потратить деньги.
  • Практическая формула: свободно внутри периметра, стоп перед необратимым.
