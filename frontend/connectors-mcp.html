Инфраструктура агента

Коннекторы и MCP-протокол

Как агент получает доступ к внешним сервисам, вызывает инструменты и остаётся в границах выданных разрешений.

Агенту нужны не только «мозги», но и подключённые руки.
Через MCP подключается всё, у чего есть программный вход — API, файлы, база данных. Не подключается то, у чего есть только «человеческий» вход — экран, мышь. Для второго нужен computer use или браузерная автоматизация.
Есть «розетка»: API / файлы / БД

Что можно подключить через MCP

  • Почта и календарь — Gmail, Outlook, Google Calendar.
  • Файлы и облака — Google Drive, Dropbox, Box, OneDrive, локальная файловая система.
  • Мессенджеры и каналы — Slack, Telegram, Discord через их API.
  • Базы данных — Postgres, MySQL, внутренняя база редакции.
  • Таск-трекеры и базы знаний — Notion, Asana, Linear, Jira.
  • Код и репозитории — GitHub, GitLab.
  • Веб-поиск и выгрузка страниц — через сервер, который даёт это как инструмент.
  • Открытые данные с API — реестры, госзакупки, статистика, карты.
  • Созвоны и транскрипты — Zoom через API.

MCP

MCP (Model Context Protocol) — универсальная «розетка» для внешних цифровых инструментов и источников данных.

  • Один формат: агенту не нужно заново учиться каждому сервису.
  • Инструменты: поиск, файлы, базы, задачи, документы.
  • Контекст: данные приходят агенту в понятной структуре.

Что нельзя подключить через MCP

Если нет программного входа, нужен другой механизм. На то есть computer use.

  • Приложение без API — например, древняя редакционная CMS: только computer use или браузер.
  • Сайт без API — браузерная автоматизация в окне, не MCP.
  • Закрытая чужая система без ключей — даже если API есть, без доступа MCP бессилен. Это вопрос разрешений, а не протокола.
  • Прямое управление экраном, мышью, скриншотами рабочего стола — это computer use, отдельный механизм.
Без MCP — тяжело

Как работает computer use

1

Скриншот

Система снимает изображение экрана или окна и отдаёт модели.

2

Зрение

Мультимодальная LLM распознаёт кнопки, поля, текст и координаты в пикселях.

3

Решение

Модель выбирает действие: кликнуть, напечатать, прокрутить, перетащить, нажать клавишу.

4

Действие

Контроллер выполняет это на реальной ОС: двигает курсор, кликает, эмулирует клавиши.

5

Снова скриншот

Агент смотрит, что изменилось, и решает следующий шаг. И так по кругу.

MCP vs computer use

MCP Структурный вход: «вызови функцию отправить_письмо с такими аргументами». Быстро, точно, надёжно.
Computer use Человеческий вход: агент действует глазами по пикселям и руками по мыши.
Цена универсальности Работает с тем, что видно на экране, но медленно и хрупко: поехала вёрстка, сдвинулась кнопка, всплыло окно.

Обязательно под разрешениями

Computer use действует на реальной машине, не в песочнице, поэтому он огорожен правами: где-то экран только «видно» (читать можно, кликать нельзя), где-то разрешён лишь клик, а перед необратимым — отправить, опубликовать, удалить — агент останавливается и спрашивает.

За computer use нужно присматривать, особенно на необратимых действиях, и не доверять ему чувствительные операции без подтверждения.
