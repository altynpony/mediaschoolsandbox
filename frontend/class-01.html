Агентность модели

Что меняется, когда LLM получает инструменты, доступы и цикл действий.

Что такое LLM

Большая языковая модель (LLM, Large Language Model) — это очень мощный предсказатель следующего слова, обученный на огромном объёме текста. Её суперспособность одна: получив текст на входе, выдать правдоподобное продолжение.

Модели, с которыми мы работаем на этом курсе — ChatGPT, Claude, Gemini — настолько большие, что их нельзя запустить на собственном компьютере. Им нужен GPU (Graphic Processor Unit) — специализированное «железо» для тяжёлых вычислений.

Что добавляет «агентность»

Агент — это та же LLM, но которой дали три вещи:

  • Руки — инструменты, tools: открыть файл, отправить письмо, выполнить поиск, запустить код.
  • Доступы — коннекторы, разрешения: почта, диск, календарь, базы.
  • Право на цикл — попробовать → посмотреть результат → исправить → продолжить, пока задача не решена.
Агент = LLM + инструменты + доступы + автономный цикл.
ЧАТ-БОТ: вы → [LLM] → ответ текстом → вы делаете руками АГЕНТ: вы → [LLM] ⟲ план → действие → наблюдение → коррекция → ... → готовый результат ↑__________ агентный цикл (agent loop) __________↑ ↓ инструменты + доступы (файлы, почта, веб, код)

Словарь

  • LLM — речь. Способность к любым языкам — и к нашим, человеческим, и к Питону.
  • GPU — нейроны. GPU — железо, на котором LLM «живёт».
  • MCP — универсальная розетка. Единый способ подключать к агенту внешние цифровые инструменты и источники данных.
  • Tool use / function calling — механизм рук. Способность модели вызвать инструмент, а не только описать его.
  • Веб, зрение, управление компьютером — другие руки. Не каждый инструмент идёт через MCP.
  • .md — язык инструкций и памяти. Простой текст с лёгкой разметкой, читаемый и человеком, и машиной.
  • Скилл ≠ память. Скилл = процедура, память = состояние и факты.
  • Контекстное окно и токены — рабочая память. Окно — сколько модель удерживает за раз.
  • Терминал ≠ ВМ ≠ песочница. Терминал — дверь, ВМ — отдельный компьютер, песочница — огороженная среда.
  • Разрешения / OAuth — ключи и красные линии. Свободно внутри периметра, стоп на красных линиях.
  • Системный промпт — должностная инструкция. Скрытый текст с ролью, правилами и границами агента.

Коннекторы и MCP-протокол

Как сервисы, данные и инструменты соединяются между собой.

Через MCP подключается всё, у чего есть программный вход — API, файлы, база данных. Не подключается то, у чего есть только «человеческий» вход — экран, мышь. Для второго нужен computer use или браузерная автоматизация.

Что можно подключить через MCP

  • Почта и календарь — Gmail, Outlook, Google Calendar.
  • Файлы и облака — Google Drive, Dropbox, Box, OneDrive, локальная файловая система.
  • Мессенджеры и каналы — Slack, Telegram, Discord через их API.
  • Базы данных — Postgres, MySQL, внутренняя база редакции.
  • Таск-трекеры и базы знаний — Notion, Asana, Linear, Jira.
  • Код и репозитории — GitHub, GitLab.
  • Веб-поиск и выгрузка страниц — через сервер, который даёт это как инструмент.
  • Открытые данные с API — реестры, госзакупки, статистика, карты.
  • Созвоны и транскрипты — Zoom через API.

Что нельзя подключить через MCP

  • Приложение без API — только computer use или браузер.
  • Сайт без API — браузерная автоматизация в окне, не MCP.
  • Закрытая чужая система без ключей — без доступа MCP бессилен.
  • Прямое управление экраном, мышью, скриншотами рабочего стола — computer use, отдельный механизм.

Computer use

Агент прикидывается человеком на уровне операционной системы. GUI — это всё, чем программа повёрнута к человеку: окна, кнопки, поля, меню. Противоположность — API, программный интерфейс: им пользуется не человек, а другая программа.

Снимок Контроллер просит ОС сделать скриншот экрана и отдаёт его модели.
Локализация Модель по картинке находит нужный элемент и его координаты в пикселях.
Команда Модель выдаёт структурный tool call: click(540, 320) или type("текст").
Впрыск Контроллер передаёт событие штатному механизму ввода ОС.
Слияние ОС вбрасывает синтетический клик в тот же поток, куда идут сигналы от настоящей мыши.
Повтор Новый скриншот → модель смотрит, что изменилось → следующий шаг.

MCP vs computer use

MCP — структурный вход: «вызови функцию отправить_письмо с такими аргументами». Быстро, точно, надёжно.

Computer use — человеческий вход: агент действует глазами по пикселям и руками по мыши. Работает с тем, что видно на экране, но медленно и хрупко.

Computer use действует на реальной машине, не в песочнице, поэтому он огорожен правами. За computer use нужно присматривать, особенно на необратимых действиях.

Cowork & Codex

Что предлагают лидеры: Cowork / Codex / Gemini.

Модель + инструменты + рабочее место с файлами: ваш компьютер, облако или оба сразу.

Claude Cowork

Cowork — режим Claude Desktop для знаний и рабочих файлов: поручить задачу, дать папку, получить готовый документ, таблицу или отчёт.

  • Работа с файлами, отчётами, папками, заметками, документами.
  • Skills — инструкции в формате .md, которые можно переиспользовать.
  • Можно отправлять задачу с телефона, но Cowork исполняется в desktop-среде.

OpenAI Codex

Codex — командный центр для агентной работы с кодом и файлами: локально, в IDE, CLI, web и desktop-приложении.

  • Репозитории, файлы, shell, browser, computer use, параллельные агенты.
  • Skills и Automations: процедуры и повторяющиеся задачи можно оформлять как рабочие сценарии.
  • Codex можно контролировать через ChatGPT mobile, включая удалённые задачи.

Google Gemini

Gemini — AI-среда Google: web, mobile, macOS app, Google ecosystem и CLI для разработчиков.

  • Связка с Google-сервисами, поиск, multimodal, Canvas, Drive/Docs-подход.
  • В CLI есть команды, @-контекст, shell mode и кастомные команды.
  • Официальный native app сейчас есть для macOS; на остальных desktop-сценариях надёжнее идти через web/CLI.

Что вызывают слэш / и собака @

  • Слэш / — команды и навыки: /schedule, /personality, /review, пользовательские процедуры.
  • Собака @ — указание файла, папки, контекста или участника.
  • Горячие клавиши — вызвать панель, открыть контекст, подтвердить действие, переключить режим или автодопуск.

Что сравниваем на занятии

  • Голосовой ввод: Codex понимает всё и всех. Cowork и Gemini даже с английским тяжело справляются.
  • Skills / инструкции: Claude и Codex особенно явно работают с .md-инструкциями.
  • Автоматизации: у Codex и Cowork есть scheduled/recurring задачи; выбирайте правильную модель.
  • Сильные стороны: Claude силён в UX и вёрстке, Gemini — в мультимодальности, Codex — лучший программист из троих и умеет рисовать.

.md Skills / Автоматизации

Как собирать, передавать и не терять редакционный опыт.

Формула: не просто делаем задачу — делаем так, чтобы из неё осталась инструкция.

Что такое .md

Markdown (.md) — простой текст с лёгкой разметкой. Он читается человеком и машиной, правится в любом редакторе, хранится где угодно, версионируется и пересылается коллеге. Агент сам читает и пишет .md.

# Заголовок 1
## Заголовок 2
- маркированный список
1. нумерованный список
**жирный**  *курсив*  `код`
[текст ссылки](https://...)
> цитата
| столбец | столбец |
``` блок кода ```

Автоматизации через MCP

Автоматизация = связка: LLM думает → коннекторы через MCP приносят и уносят данные → расписание запускает без вас.

  • Cowork Routines: по расписанию в облаке, работают даже при закрытом ноутбуке.
  • Codex Automations: расписания и триггеры — поминутно, ежедневно, сложные паттерны.
  • Будни 8:00 — агент собирает упоминания за ночь и кладёт сводку в общую папку к планёрке.

Создание собственных скиллов

Каждую повторяемую задачу можно постепенно превратить в Skill. Можно взять готовый шаблон, но важнее уметь извлекать свой: пройти задачу один раз с агентом и в конце попросить описать процесс как SKILL.md.

Сделали Задачу один раз.
Поняли Шаги и правила.
Зафиксировали Входы, запреты, формат.
Сгенерировали Агент пишет SKILL.md.
Отредактировали Человек проверяет.
Повторили Применяем снова.
---
name: design-critique
description: когда применять — "review this design", скриншот, ссылка на Figma...
---
# тело: что нужно на входе → процедура → шаблон вывода

Системные соглашения

Файл Для кого Что это
CLAUDE.md Claude Самый распространённый файл инструкций проекта.
AGENTS.md Codex и другие агенты Открытый вендоронезависимый стандарт; попытка сделать один файл, понятный любому агенту.

SKILL.md Стандарт скилла Описание одного навыка: когда включать, что делать, какой результат вернуть.

Библиотеки скиллов

Скиллы не пишут с нуля каждый раз. Их берут из библиотек, публикуют свои и обмениваются с коллегами — как плагинами.

  • Папка агента может хранить инструкции, скиллы и память.
  • Общая папка помогает команде подхватить один и тот же накопленный опыт.
  • .md-формат не привязан к одному вендору.

Версионирование

Агент правит файлы на месте. Если разрешить крупное изменение без снимков, можно потерять хорошо сделанную работу.

  • Перед крупным изменением сохраняй версию / делай чекпоинт.
  • Сохрани текущую версию отдельным файлом, прежде чем переписывать.
  • Держите проект там, где есть история версий: Git, Dropbox, облачный диск.

Облако vs ваш диск

Где лежит Плюс Минус Когда выбирать
Локальный диск Приватность, выбранная папка, контроль. Работает, пока компьютер включён. Черновики, чувствительные источники, внутренняя работа.
Облако Доступно отовсюду, удобно для команды, работает в фоне. Нужны доступы и доверие к сервису. Общие материалы, планёрки, командные папки.

Сквозной редакционный кейс

Задача: каждое утро готовить медиа-дайджест по теме X и складывать в общую папку команды.

  1. Recurrent task запускает агента в 7:30.
  2. LLM читает инструкцию из скилла .md.
  3. Через MCP-коннектор агент собирает свежие материалы.
  4. LLM фильтрует, реферирует и группирует по сюжетам.
  5. Результат сохраняется как .md/документ в облачную папку команды.
  6. В память агент пишет, что уже показывал, чтобы завтра не повторяться.

Примеры

  • Разметка документов.
  • Мониторинг.
  • Адаптация материалов.
  • Fact-checking ассистент.
  • Разбор информации.
  • Repack.
  • Поиск по собственному архиву.

Практический маршрут

  1. Найти готовый скилл, если задача типовая.
  2. Нет готового — пройти задачу один раз с агентом.
  3. Попросить: «опиши этот процесс как SKILL.md».
  4. Отредактировать вручную.
  5. Сохранить в проекте / библиотеке скиллов.
  6. Использовать при следующем похожем запросе.
  7. После каждого сбоя — улучшать скилл.
Агент ускоряет рутину, но ответственность за факты, источники и этику остаётся на человеке. Любой вывод агента — черновик.

Сценарии и агенты

Как появляются помощники под задачи редакции.

От любопытства к плану

Кейс показывает рабочий маршрут: не “напиши код”, а сначала понять интерес, сформулировать запрос, собрать материалы и превратить всё в исполнимый план.

Любопытство Начинаем с живого вопроса: что можно автоматизировать, показать студентам или превратить в маленький инструмент.
Промпт Формулируем задачу для агента: контекст, аудитория, желаемый результат, ограничения и формат ответа.
Материалы Складываем в диалог заметки, HTML, markdown, ссылки, черновики и уточнения.
План Получаем исполнимый план: шаги, файлы, проверки, публикация и следующий вопрос.

От задачи к skill

Кейс с созданием видео показывает, как разовая просьба превращается в повторяемую инструкцию: что спросить, какие материалы собрать, как проверить результат и где остановиться перед публикацией.

Задача Сформулировать, какое видео нужно: аудитория, длительность, формат, исходники, ограничения.
Процедура Разложить работу на шаги: сценарий, подбор материалов, монтажные решения, титры, проверка.
Skill Сохранить повторяемый рецепт в .md, чтобы агент мог выполнять похожие видео-задачи снова.
